# LLM Wiki 사용자 가이드

LLM Wiki는 다양한 문서(Markdown)와 지식 그래프(Knowledge Graph)를 결합하여 AI 기반의 지식 관리 및 탐색을 돕는 도구입니다. 이 문서는 앱의 주요 기능과 사용 방법을 설명합니다.

---

## 1. 시작하기: 프로젝트 선택 및 생성
앱을 실행하면 가장 먼저 프로젝트를 선택하거나 생성해야 합니다.

- **프로젝트 선택**: 원격 Git 저장소(`claude-skills`)의 브랜치 목록이 프로젝트 리스트로 출력됩니다. 원하는 프로젝트(브랜치)를 선택하여 작업을 시작할 수 있습니다.
- **프로젝트 생성**: 새로운 프로젝트 이름을 입력하여 생성하면, 원격 저장소에 해당 이름의 브랜치가 생성됩니다. 데이터 격리를 위해 FalkorDB 내에서 프로젝트 이름을 접두사(prefix)로 사용하는 전용 그래프 공간이 생성됩니다.
- **Git 인증**: 환경 설정(`.env`)에 정의된 Git 사용자 정보가 인증에 사용됩니다.

## 2. 문서 및 데이터 관리 (Knowledge Ingestion)
이 시스템은 Markdown 기반의 문서를 주 데이터 소스로 사용합니다.

- **Markdown 변환**: PDF, Word, HTML 등 다양한 포맷의 문서를 앱에 추가하면 자동으로 Markdown(`.md`) 파일로 변환되어 `raw/sources/` 폴더에 저장됩니다.
- **문서 편집**: 내장된 에디터를 통해 문서를 직접 수정하거나 관리할 수 있습니다.

## 3. 지식 그래프 (Knowledge Graph)
문서 내의 정보를 연결하여 시각화하고 AI가 참고할 수 있는 지식 그래프를 생성합니다.

- **AI 기반 그래프 생성**: 문서를 분석하여 자동으로 노드(Node)와 관계(Edge)를 추출합니다.
- **멀티 그래프 구조**: 하나의 거대한 그래프 대신 `ui_graph`, `skill_graph`와 같이 성격에 따라 작은 단위의 그래프들로 나누어 관리합니다.
- **시각화 및 CRUD**: 
  - 그래프 뷰어를 통해 지식의 연결 구조를 시각적으로 확인할 수 있습니다.
  - 노드, 관계, 속성을 직접 추가, 수정, 삭제(CRUD)할 수 있습니다.
- **문서-그래프 연관**: 특정 문서를 선택하면 해당 문서와 관련된 지식 그래프 목록이 표시되며, 클릭 시 해당 그래프로 바로 이동합니다.

## 4. 리뷰 및 스키마 검증
AI가 새로운 지식(노드 타입, 관계 타입 등)을 생성하려고 할 때, 사용자의 승인이 필요할 수 있습니다.

- **스키마 변경 승인**: AI가 기존에 없던 새로운 관계 유형이나 속성을 제안하면 **Review** 탭에 알림이 표시됩니다.
- **Approve/Reject**: 사용자는 제안된 변경 사항을 확인하고 승인(Approve)하거나 거절(Reject)할 수 있습니다. 거절된 타입은 'Forbidden' 리스트에 등록되어 이후 생성 시 제외됩니다.

## 5. AI 채팅 및 질문 타입 (Q&A)
LLM에게 질문을 던져 문서와 지식 그래프를 바탕으로 답변을 얻을 수 있습니다.

- **RAG 및 그래프 참고**: AI는 검색(RAG) 기능과 구축된 지식 그래프를 동시에 참고하여 정확한 답변을 생성합니다.
- **질문 타입(Question Types)**:
  - 질문 창 하단에서 특정 질문 타입(예: Balance Simulation, Risk Recommendation 등)을 선택할 수 있습니다.
  - 각 타입은 **YAML** 또는 **Markdown** 템플릿으로 정의되어 있으며, AI는 해당 양식에 맞춰 구조화된 답변을 제공합니다.
  - 구조화된 답변은 가독성 좋은 리스트 형태로 화면에 렌더링됩니다.

## 6. 승격(Promotion) 및 배제(Exclusion) 지도
검색 효율을 높이기 위해 불필요한 데이터를 필터링하는 규칙을 관리합니다. 이 기능은 사이드바의 **Promotion(아이콘: TrendingUp)** 탭에서 제공됩니다.

- **배제 설정(Exclusions)**: 특정 질문 유형에서 검색 결과에 포함되지 않아야 할 경로(path)나 패턴을 설정합니다.
  - **Pattern**: 특정 질문 유형에만 적용되는 규칙입니다.
  - **Axiom**: 모든 질문 유형 또는 여러 유형에 공통으로 적용되는 전역 규칙입니다.
- **승격(Promotion)**: 실제 검색 과정에서 반복적으로 배제된 항목(Instance)을 분석하여, 이를 정식 규칙(Pattern 또는 Axiom)으로 확정하는 과정입니다. 사용자의 승인을 통해 지식 베이스의 검색 정확도를 점진적으로 개선합니다.
- **신선도 검사(Freshness Check)**: 원본 파일이 변경되었을 때 기존 배제 규칙이 여전히 유효한지 검사하여 재검토가 필요한 항목을 표시합니다.

## 7. 설정 및 커스터마이징
앱의 동작 방식을 세밀하게 조정할 수 있습니다.

- **질문 타입 CRUD**: `Settings > Question Types` 메뉴에서 질문 템플릿을 직접 추가, 수정, 삭제할 수 있습니다.
- **FalkorDB 설정**: `Settings > FalkorDB`에서 연결 URL을 설정하고 연결 테스트를 수행할 수 있습니다.

## 8. 버전 관리 및 동기화
모든 작업 내용은 Git을 통해 안전하게 관리됩니다.

- **로컬 작업**: 문서 수정 및 지식 그래프 변경 사항은 우선 로컬에서 관리됩니다.
- **원격 동기화**: 앱을 종료하거나 다른 프로젝트로 전환할 때, 변경 사항을 원격 브랜치에 반영할지 묻는 팝업이 뜹니다. 프로젝트 선택 시 항상 최신 상태를 유지하기 위해 원격과 동기화(Sync)를 시도합니다.

---

**팁**: 복잡한 질문을 할 때는 적절한 **Question Type**을 선택하는 것이 훨씬 더 유용한 답변을 얻는 지름길입니다!

# 구현 전 기획 결함 탐지

구현에 들어가기 전에 규칙의 모호함, 누락된 우선순위, 서로 충돌하는 의도, 플레이어가 오해할 수
있는 설계 결함을 찾는다. "이건 애매한데?"라고 사람이 리뷰에서 발견해야 할 지점을 미리 드러내는
질문 유형이다.

## Input

- 제안된 기획안 또는 변경안
- 관련 시스템·콘텐츠·정책 범위
- 반드시 보존해야 하는 의도 또는 플레이 경험

## Output

- 모호하거나 충돌하는 문장·규칙 인용
- 구현 전에 결정해야 할 질문 목록
- 결함을 줄이기 위한 명세 보강 제안

## Zero residue

잔존 0 = **기획 결함 신호 없음**. 현재 wiki 기준으로 구현을 막을 만한 모호함이나 충돌이
발견되지 않았다는 신호.

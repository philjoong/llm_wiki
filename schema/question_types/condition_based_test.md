# 조건 기반 가상 테스트

특정 조건/상호작용에서 시스템이 어떻게 동작해야 하는지 가상의 테스트 케이스로 검증한다.
규칙·정책·시스템 명세를 인용해 결과를 추론하며, 실제 게임 빌드에서 확인하기 어려운 엣지 케이스가
주된 대상이다.

## Input

- 트리거 조건 (어떤 상황·상태에서)
- 행위 (어떤 동작·입력이 일어나면)
- 기대 결과 (시스템이 어떻게 반응해야 하는가)

## Output

- 인용된 규칙/정책/시스템 페이지
- 조건에 적용 가능한 결과 한 줄 또는 명세 공백

## Zero residue

잔존 0 = **명세 공백**. 이 조건에 대한 규칙이 wiki 어디에도 정의되어 있지 않다는 신호.
"결과 없음"이 아니라 "사람이 정의해야 한다"는 정상적인 출력이다.

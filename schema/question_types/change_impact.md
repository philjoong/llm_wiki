# 변경 영향 분석

특정 수치·공식·정책·콘텐츠 변경이 다른 시스템·콘텐츠에 어떤 영향을 주는지 추적한다.
회귀 테스트(`regression_test`)가 "깨질지 모름"이라면, 변경 영향은 "어디까지 의도된 변화인가"를 본다.

## Input

- 변경 대상 (어떤 페이지·수치·정책)
- 변경 종류 (수치 변경 / 정책 강화 / 신규 추가 / 폐기)

## Output

- 영향받는 페이지 목록 (related: 연결 + 출처 인용을 따라간 결과)
- 동시 갱신이 필요한 페이지 후보

## Zero residue

잔존 0 = **영향 격리 완료**. 이 변경이 다른 페이지로 전이되지 않는다는 신호.
독립적으로 적용 가능하다.

# 회귀 테스트

이전 패치/변경 이후 깨질 가능성이 있는 영역을 식별한다.
liveops/patches/ 와 변경된 시스템·콘텐츠·밸런스 페이지 간의 의존성을 따라가며,
지난 패치가 건드린 인접 영역을 회귀 후보로 본다.

## Input

- 변경 대상 (어떤 시스템·콘텐츠·수치가 바뀌었는가)
- 변경 범위 (수치/공식/로직/UI 중 무엇)
- 기존 회귀 스위트 (이미 정의된 테스트 케이스가 있는가)

## Output

- 회귀 후보 페이지 (영향받을 가능성이 높은 시스템·콘텐츠·QA 케이스)
- 적용 가능한 기존 회귀 케이스 인용

## Zero residue

잔존 0 = **영향 없음**. 변경된 영역이 다른 시스템·콘텐츠와 격리되어 있으며 회귀 위험이 없다는 신호.

# 테스트 공백 탐지

기능·시스템·콘텐츠 중 QA가 누락되었거나 회귀 케이스가 정의되지 않은 영역을 식별한다.
회귀 테스트(`regression_test`)와 달리, 변경된 영역이 아니라 **테스트 자체가 빈 영역**을 본다.

## Input

- 대상 영역 (어떤 시스템·기능·콘텐츠인가)
- 기존 QA 자산 (test_plans / regression_suites / known_issues에 무엇이 있는가)

## Output

- QA 자산이 비어 있는 페이지 목록
- 신규로 정의해야 할 케이스 후보

## Zero residue

잔존 0 = **공백 없음**. 대상 영역의 모든 시스템·기능에 대응하는 QA 자산이 존재한다는 신호.
